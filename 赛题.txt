赛题背景
高性能服务器处理很多业务的CPU消耗很高，例如全闪存的存储服务器，随着NVME盘的性能迅速提升，仅仅处理IO本身就已经对CPU消耗巨大，这使得如果在服务上面还要跑别的业务，例如存储之上跑数据库，数据库之上跑应用或者中间件的CPU资源就很紧张了，而且因为CPU相互竞争影响性能。希望通过GPU offload 部分计算密集型的任务，减小主CPU的资源消耗。

赛题任务
通过 dm 在 block 层作压缩
优点，上层支持各种文件系统

通过 btrfs、zfs 在文件系统层作压缩
优点，内容感知，有可能支持更好的压缩效率，但是被锁定在特定文件系统了，而这些文件系统本身的绝对性能可能不如 xfs、ext4 这些

性能指标可以根据网上相关测试为基准，然后实测，接着看 GPU offload 的性能改进

纯 CPU 实现的性能和 CPU、DRAM 开销
GPU offload 实现的 CPU、DRAM 开销
CPU 比较不同架构，x86、aarch64 等的性能

fio，vdbench 测试性能
top，htop，perf，stat，sar 等工具分析资源消耗

赛题特征
GPU offload 对国产化 CPU 处理计算密集型任务稍弱的补充

评审要点
1. 创新性
英伟达最新一代 GPU 自带了压缩引擎（https://developer.nvidia.com/nvcomp）。但更加突出 GPU 通用算力处理压缩解压任务，提供了更广泛的适配性，不锁定最新 GPU，不锁定 GPU 供应商，需要更深入了解 GPU 的特点和性能优化方式。

跑起来，有完整性能测试报告，测试用例的覆盖率相对完整，文档描述清楚，根据文档可以完成部署和二次开发与集成。
